## What's new

Meowdo is a fast, private to-do list in your browser toolbar — organize tasks, see what's left,
and get optional reminders. Everything stays on your device.

- **Quick capture** — type a task and hit Enter; the toolbar **badge** shows how many tasks are
  still left across all lists.
- **Lists** — group tasks into as many lists as you like; switch with the dropdown or view **All
  lists**; rename/delete from the ⋯ menu.
- **Check it off** — one click to complete (drops to the bottom, struck through); **filter** by
  All / Active / Done; **clear completed**.
- **Edit in place** — click a task to rename it; clearing the text deletes it. No popups.
- **Optional reminders** — pick from a dropdown (**In 30 minutes**, **1 hour**, **2 hours**,
  **This evening**, **Tomorrow**) or **Custom…** for an exact date/time, with a live "→ time"
  hint. When a reminder is due you get **three cues**: a system **notification** (with **Mark
  done** / **Snooze 10 min**), a prominent on‑screen **alert window**, and a synthesized
  **chime** (consistent on Windows/Linux/macOS). Reminders fire even if the browser was closed
  when they came due.
- **Private by design** — no network, no analytics, no remote code; tasks live only in
  `chrome.storage.local`.

## Under the hood — shipped files

| File | Role |
| --- | --- |
| `manifest.json` | MV3 manifest (action popup, background service worker, four permissions). |
| `store.js` | Shared data model + `chrome.storage.local` (loaded by the popup, worker, and alert). |
| `popup.html/.css/.js` | The surface: add / edit / complete / delete tasks, lists, filters, reminder editor. |
| `background.js` | Service worker: the toolbar badge + reminder alarms → notification + chime + alert window. |
| `offscreen.html/.js` | Audio-only document that synthesizes the reminder chime (Web Audio; no file). |
| `alert.html/.css/.js` | The on-screen reminder alert window (Mark done / Snooze / Dismiss). |
| `icons/icon{16,32,48,128}.png` | Toolbar/store icons (procedurally generated cat-checklist). |

Dev/build files (`build.sh`, `icons/gen_icons.py`, docs) are excluded from the zip.

## Permission justifications (paste-ready for the dashboard)

| Permission | Why Meowdo needs it |
| --- | --- |
| `storage` | Save the user's tasks, lists, and settings locally (`chrome.storage.local`). No data leaves the device. |
| `alarms` | Fire a reminder at the chosen time, including when the browser was closed at that moment. |
| `notifications` | Show the reminder notification (Mark done / Snooze) when a task comes due. |
| `offscreen` | Play the reminder chime via the Web Audio API. Audio only — no network, no remote code. |

**Single purpose:** "A to-do list: create tasks, organize them into lists, and set optional
reminders that fire a notification, an on-screen alert, and a chime when a task is due — entirely
on the user's device."

## Verification (run before submitting)

Load the unpacked extension (`chrome://extensions` → Developer mode → Load unpacked) and confirm:

1. **Add / complete / delete** a task; the toolbar **badge** tracks the count of what's left.
2. **Edit in place** (click text, Enter/Esc); clearing the text deletes the task.
3. **Filters** (All / Active / Done) show the right subset with correct counts.
4. **Lists** — ＋ add, dropdown switch, **All lists**, ⋯ rename, ⋯ delete (asks once).
5. **Reminders** — set a **Custom…** time ~1 min out; at due time you get the **notification**,
   the **alert window**, and the **chime**; **Mark done** and **Snooze 10 min** both work (from
   the notification and the alert window).
6. **Persistence** — reopen the popup and reload the extension; tasks, lists, and reminders
   survive. A reminder set for a past minute fires on the next worker start.
7. **No network** — with DevTools open on the popup and the service worker, exercise every
   feature and confirm zero outbound requests.

## Known limitations

- **One device.** Tasks live in `chrome.storage.local`, so they don't sync across machines (by
  design — nothing leaves the device).
- **The notification banner is the OS's job.** If Chrome notifications are disabled at the OS
  level the banner won't appear — the badge, the on-screen **alert window**, and the **chime**
  still work. The notification *sound* also depends on the OS (Windows plays one automatically,
  macOS depends on the per-app setting, Linux is often silent), which is why Meowdo plays its own
  chime.
- **Reminders need the machine awake.** A sleeping computer delays the alarm until it wakes.
- Reminder times use the computer's local timezone.
